# SP1.5: migrate host-side guest callers from the legacy JSON protocol to gRPC

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Move every host-side caller of the guest agent off the legacy JSON vsock protocol (`vsock.AgentPort` = 52) onto the `sandbox.v1` / `controlv1` gRPC contract (`AgentGRPCPort` = 53), so the gRPC-only Rust agent (#329) becomes a true production drop-in and the legacy JSON loop becomes dead code that can be removed.

**Architecture:** #326 already built the host-side gRPC-over-vsock client infrastructure (the GuestConn-over-gRPC bridge, the vsock gRPC dialer used by the Connect/sandboxrpc service). SP1.5 rewires the remaining direct JSON callers to use that same gRPC client path instead of `vsock.Connect(.., vsock.AgentPort)` + the JSON request/response types. No new protocol; reuse the gRPC client the Connect service already uses.

**Tech Stack:** Go. The host-side gRPC client over vsock (from #326). `proto/sandbox/v1` + `proto/sandbox/controlv1` generated clients.

## Global Constraints

- Branch off main AFTER #326 has merged to main (sentinel: `git show origin/main:internal/vsock/protocol.go | grep -c AgentGRPCPort` > 0). Branch name: `feat/sp1.5-json-to-grpc-host-callers`.
- No em (U+2014) or en (U+2013) dashes anywhere. DCO `git commit -s`. Conventional commits. Stage explicit paths only.
- Each caller migration must keep the existing behavior (fork-readiness wait, exec, file IO) byte-equivalent over gRPC; the Go `firecracker-test` / `kind-e2e` / smoke CI must stay green at each step.
- The legacy JSON loop in the Go agent + the JSON protocol code are NOT removed in SP1.5; they are made unused. Removal is a later step (after the Rust default-flip soaks).
- Verify each migrated caller against BOTH agents where feasible (the Go agent serves both 52 and 53; the Rust agent serves only 53). After SP1.5, the Rust agent (#329) must pass the full firecracker-test / e2e as `/init` (AGENT_IMPL=rust).
