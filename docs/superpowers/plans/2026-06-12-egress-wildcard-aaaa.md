# Guest Egress: Wildcard Names and AAAA/IPv6 (#3b / #47)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Extend the name-based egress allowlist (the controlled DNS resolver, #47, `internal/dnsproxy`) from exact-match A-only to (1) suffix WILDCARD names (`*.anthropic.com` allows any subdomain) and (2) AAAA/IPv6 answers (A-only today). The DNS proxy resolves an allowed name, pins the resolved address into the per-sandbox nftables allow set for the record TTL, and is the guest's only resolver; this slice makes the allowlist more expressive without widening the trust model (still allow-by-name, default-deny). Verified: exhaustive matcher unit tests including the BYPASS cases (a wildcard must never allow the bare domain, a non-subdomain, or a look-alike), AAAA pinning into the v6 nftables set, and a security review of the wildcard matcher (an over-broad wildcard is an egress escape, so the matcher correctness is the load-bearing property).

**Honesty constraint:** the wildcard matcher is a security boundary. `*.example.com` MUST match `a.example.com` and `a.b.example.com` (any subdomain) and MUST NOT match: `example.com` (the bare apex, no subdomain label), `notexample.com` / `evilexample.com` (not a subdomain, no leading dot boundary), `example.com.evil.com` (the allowed name is not the suffix), or `a.example.com.evil.com`. The anchor is the leading dot: a name matches `*.D` iff it ends with `.D` and has at least one non-empty label before it. Case-insensitive, FQDN-normalized (trailing-dot tolerant) like the existing exact matcher. No regex from user input (a literal suffix check, not a compiled pattern, to avoid ReDoS and accidental over-match). AAAA support is additive (A still works); if only A or only AAAA is allowed by the resolver answer, pin what resolved, do not fabricate the other family. The default stays deny; a name not matching any exact or wildcard entry resolves to nothing (NXDOMAIN/refused) and pins nothing. The threat-model egress row is updated in the same PR (the allowlist is now wildcard+v6 capable; the matcher anchor rule is the stated guarantee).
